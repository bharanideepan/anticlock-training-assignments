# Implementation Plan: Detail Page Enrichment

**Branch**: `009-detail-page-enrichment` | **Date**: 2026-06-14 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/009-detail-page-enrichment/spec.md`

## Summary

Replace the generic `DataList` component used across all detail page tabs with rich, typed row components that show all relevant fields per record type. Wire up the Opportunity detail Tasks/Files tabs (currently showing placeholder text) to real API data. Fix the Contact detail Opportunities tab (currently a placeholder) and enrich the contact summary panel with clickable mailto/tel links. Add "Add" action buttons to each tab, gated by existing role permissions. Implement lazy tab loading via React Query `enabled` prop. All changes are frontend-only — no backend or schema changes required.

## Implementation Strategy

### Phase 1 — Shared Components (Blocking Prerequisite)

Create all 6 shared components before updating any detail page. These are pure presentational — no API calls, no hooks.

### Phase 2 — Customer Detail Page (MVP / Highest Priority)

Replace `DataList` with typed row components. Add lazy loading via `enabled: tab === N`. Pass `{ pageSize: 10 }` to all tab hooks. Add "Add" buttons and "View all" links.

### Phase 3 — Opportunity Detail Page

Wire Tasks tab to `useTasks({ opportunityId, pageSize: 10 })`. Wire Files tab to `useFiles('OPPORTUNITY', id, enabled)`. Open `TaskFormDialog` on "Add Task".

### Phase 4 — Contact Detail Page

Enrich summary (mailto/tel links with icons). Fix Opportunities tab with `useOpportunities({ contactId, pageSize: 10 })`. Enrich Activities tab. Open `ActivityFormDialog` on "Log Activity".

### Phase 5 — Polish

`pnpm tsc --noEmit`, remove `DataList` dead code, verify no placeholder text remains.
