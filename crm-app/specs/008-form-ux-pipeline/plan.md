# Implementation Plan: Form UX & Pipeline Improvements

**Branch**: `008-form-ux-pipeline` | **Date**: 2026-06-14 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/008-form-ux-pipeline/spec.md`

## Summary

Replace raw UUID text fields in all CRM forms with searchable, paginated Autocomplete dropdowns. Fix the global search bar crash caused by an invalid listbox slot prop. Fix pipeline drag-and-drop z-index by adding a `DragOverlay`. Add color coding to pipeline stage columns. All changes are purely frontend — no backend schema or API changes required.


**Key behaviors**:
- `ListboxProps={{ onScroll: handleScroll }}` where `handleScroll` checks `scrollTop + clientHeight >= scrollHeight - 50` to trigger next page
- `filterOptions={(x) => x}` — disable client-side filtering (server does it)
- `isOptionEqualToValue={(opt, val) => opt.id === val.id}` — compare by UUID
- `getOptionLabel={(opt) => typeof opt === 'string' ? opt : opt.label}` — freeSolo compat

### Pipeline DragOverlay Pattern

```typescript
// In PipelineBoardPage:
const [activeCard, setActiveCard] = useState<ActiveCard | null>(null);

<DndContext
  onDragStart={(event) => setActiveCard({ id: event.active.id, ...event.active.data.current })}
  onDragEnd={(event) => { handleDragEnd(event); setActiveCard(null); }}
  onDragCancel={() => setActiveCard(null)}
>
  {/* columns */}
  <DragOverlay>
    {activeCard && <OpportunityCardOverlay {...activeCard} />}
  </DragOverlay>
</DndContext>
```

In `OpportunityCard.tsx`, when `isDragging === true`, render the card with `visibility: hidden` (not `opacity: 0`) to preserve layout space while the overlay floats above.
